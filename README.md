Temat Projektu:  

Testowanie podatności aplikacji webowej. 

Projekt polega na analizie bezpieczeństwa aplikacji Juice Shop przy użyciu narzędzi do testowania. 

 

Plan działania projektu: 

1. Przygotowanie środowiska 

Instalacja i konfiguracja potrzebnych narzędzi. 

Sprawdzenie poprawnego działania aplikacji. 

Konfiguracja przeglądarki z proxy ZAP. 

 

2. Zapoznanie się z narzędziami 

Krótkie poznanie OWASP Top 10. 

Nauka podstaw OWASP ZAP.  

Testowe skanowanie dowolnej strony. 

 

3. Testy podatności  

Wykonanie testów OWASP ZAP. 

 

4. Analiza wyników 

Przegląd raportów ZAP. 

Ocena ryzyka. 

 
5. Propozycje zabezpieczeń 

Jak naprawić wykryte podatności i ewentualne wdrożenie poprawek


 
6. Opracowanie raportu 

 

Harmonogram pracy: 

Tydzień 1: Zapoznanie z technologia i wykorzystywanymi narzędziami oraz ich instalacja.  

Tydzień 2: Przeprowadzenie testów podatności oraz analiza wyników. 

Tydzień 3: Propozycja naprawy wykrytych błędów. 

Tydzień 4: Przygotowanie dokumentacji.
